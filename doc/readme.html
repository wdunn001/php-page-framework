some notes:
the page relies on a url rewrite in the web.config to redirect links.
and .php file can be called with a link with just the filename and not the extension it will automaticly insert the page title (exchanging _ for spaces and add .php when looking for the file.

This site is the equivalent to the asp.net site master system and works in the same general way.

whats actually happening is the index.php is always being called and the accompanied php files are loaded into a get so like home is actually index.php?page=home.php when inserted into a link.

SUPER IMPORTANT:
external links have to be fully fleshed so like www.contosco.com isnt going to cut it its gotta be http://www.contosco.com/ or the rewrite might try to load it in a variable. for local resources this isnt an issue as long as they are in a tag. I.E. img src=blah/blah/foo.jpg

the index sets variables and fixes the title and includes the master the site master includes a header and a footer and menu plus whatever page is called
